Short presentation

NeuroApp is an app that recognise lung diseases using computational visualisation over lung x-rays. The complexity of the app consinsts on neural networks and deep learning. In order to make the app user friendly, I've decided to create a simple user interface that presents the purpose of the app and guides the user through the possible use-case-uri.

WireFrames

The main page contains a short description of the application and the possibiliy to log in.

The user page contains information about the user, anterior results and a button used for uploading photos to the test of the neural network.

The results page contains information about the pacient, a short result and some statistics.

Possible questions

Can this app replace the opinion of a medical specialist?

No, it can not. To be sure that every user knows this, each page contains on bottom a warning part. Also, the ui has two blue colored lines that separates the components of the design.( It is easier for users to pay attention to a specific component when that component is isolated from all the design.)

How does a user know how to use the app?

Even from the first page, all the important parts are indicated by blue components. Also, there are descriptions that help users interact.

How can users read and understand the results of the app?

This part may be difficult for some users but with the help of three different data visualisation it becomes easier. They simply go to the results page and click on visualisations.
